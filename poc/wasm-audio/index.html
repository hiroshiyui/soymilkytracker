PoC: Rust WASM → Web Audio AudioWorklet

Proof-of-concept spike for SoymilkyTracker.
Rust/WASM creates an AudioContext, loads a SineProcessor as an AudioWorkletNode, and connects it to the audio output.

Status: idle — click to start (browser requires a user gesture)

The SineProcessor runs in the browser's dedicated audio thread.
In the full app, this processor will be replaced by a WASM module running the tracker engine (fundsp DSP + oxisynth), driven from the main thread via MessagePort.
